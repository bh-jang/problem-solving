 ## 문제
 [
    [5, 3, 7, 2, 3],
    [3, 7, 1, 6, 1],
    [7, 2, 5, 3, 4],
    [4, 3, 6, 4, 1],
    [8, 7, 3, 5, 2],
];

가 있으면 '상하좌우'가 현재위치보다 낮으면 봉우리라고 한다.
봉우리의 개수를 구하라. '상하좌우'에 값이 없으면 0이라고 한다.

## 풀이(1)
- index가 초과하는 경우(maxRow, maxCol보다 크거나, 0보다 작아지면)는 자동 true
- 상하좌우값을 현재 포지션에서 비교만하여 처리
- if문이 너무 길다.

## 풀이(2)
- 탐색할 path를 dx, dy로 미리 만들고 처리하면 코드가 좀 더 가독성이 좋다.
